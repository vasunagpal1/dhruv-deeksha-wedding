Part 1

Connect this project to GitHub

  1. Create the GitHub repository in your browser

    Go to github.com and make a new repository.

    • Choose a name like `dhruv-deeksha-wedding`.
    • Choose `Public` if you want GitHub Pages to be simple.
    • Do not add a README, `.gitignore`, or license on GitHub.

  2. Set your Git identity on this Mac

    Run this once if Git has not been configured yet.

    Terminal 
    git config --global user.name "Your Name"
git config --global user.email "you@example.com"

  3. Create the first commit

    Run this inside this project folder.

    Terminal 
    git add .
git commit -m "Initial wedding invitation site"

  4. Connect the local folder to your GitHub repository

    Replace the example URL below with your actual repository URL from GitHub.

    Terminal 
    git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO.git
git push -u origin main

    If Git says `origin already exists`, use this instead:

    Terminal 
    git remote set-url origin https://github.com/YOUR-USERNAME/YOUR-REPO.git
git push -u origin main

Part 2

Host the website using GitHub Pages

  1. Open your repository on GitHub

    Once the push is complete, open the project repository in your browser.

  2. Turn on GitHub Pages

    • Click `Settings`.
    • Click `Pages` in the sidebar.
    • Under `Source`, choose `Deploy from a branch`.
    • Select branch `main`.
    • Select folder `/ (root)`.
    • Click `Save`.

  3. Wait for GitHub Pages to publish the site

    GitHub usually gives you a live URL like `https://YOUR-USERNAME.github.io/YOUR-REPO/`.

  4. Open the live URL and test it

    Check the page on desktop and phone after the deployment finishes.

Important: `index.html` must stay in the root folder for GitHub Pages to serve the site correctly.

Part 3

How to add the real couple images

Permanent live website method

This is the correct method for the final public wedding website.

  • Choose your first main portrait.
  • Choose your second portrait.
  • Rename them exactly to `dhruv-deeksha-1.jpg` and `dhruv-deeksha-2.jpg`.
  • Place them in the `assets` folder and replace the current files.
Terminal 
git add assets/dhruv-deeksha-1.jpg assets/dhruv-deeksha-2.jpg
git commit -m "Add final couple portraits"
git push

Temporary preview method

This is only for testing in your own browser.

  • Open the invitation website.
  • Click `Load your portraits`.
  • Use `Portrait one` and `Portrait two` to upload images or paste public image URLs.
  • The page updates instantly, but only in that browser.
This does not publish the photos to GitHub. For the final live site, replace the real image files in `assets/`.

Part 4

How to update the website later

Any time you change text, photos, colors, or sections, use the same three commands:

Terminal 
git add .
git commit -m "Describe your update"
git push

GitHub Pages will redeploy automatically after each push.

Troubleshooting

If the site does not show properly

  • Make sure `index.html` is still in the root folder.
  • Make sure GitHub Pages is using `main` and `/ (root)`.
  • Make sure your latest changes were pushed with `git push`.
  • Wait a few minutes and refresh the `Pages` settings tab.
  • Open the repository `Actions` tab if GitHub shows a deployment problem.
